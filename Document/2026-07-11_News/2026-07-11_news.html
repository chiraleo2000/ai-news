Meta ถอดฟีเจอร์สร้างคอนเทนต์ AI จากบัญชีสาธารณะบน Instagram หลังกระแสตอบรับด้านลบ

Meta ประกาศยกเลิกฟีเจอร์สร้างคอนเทนต์ด้วย Meta AI บน Instagram ที่เพิ่งเปิดตัวไปไม่กี่วันก่อนหน้า เมื่อวันที่ 11 กรกฎาคม 2026 หลังกระแสตอบรับจากผู้ใช้เป็นไปในทางลบอย่างหนัก ฟีเจอร์นี้ใช้โมเดลสร้างภาพ Muse Image ให้ผู้ใช้พิมพ์ @ เพื่ออ้างอิงโพสต์จากบัญชีสาธารณะใดก็ได้บน Instagram มาสร้างเนื้อหาต่อด้วย AI โดยเจ้าของบัญชีสาธารณะต้องปิดกั้นการเข้าถึงเอง (opt-out) ทำให้เกิดกระแสวิพากษ์วิจารณ์รุนแรงในโซเชียลมีเดีย มีโพสต์สอนวิธีปิดไม่ให้ Meta AI เข้าถึงเนื้อหาของตนจำนวนมาก และหลายองค์กรด้านงานสร้างสรรค์รวมถึงนักแสดงในสหรัฐออกมาโจมตีฟีเจอร์นี้ โดยมองว่า Meta ผลักดันฟีเจอร์จนมองข้ามประเด็นการละเมิดสิทธิเนื้อหาของผู้ใช้ Meta จึงตัดสินใจถอดฟีเจอร์นี้ออกจากระบบทันที สะท้อนความอ่อนไหวของสาธารณะต่อการใช้ AI ดึงเนื้อหาคนอื่นมาสร้างสรรค์ใหม่โดยไม่ได้รับความยินยอมชัดเจน

ผลกระทบ: กระแสต่อต้านฟีเจอร์ AI ของ Instagram ชี้ให้เห็นความเสี่ยงด้านความยินยอมและสิทธิเนื้อหาเมื่อแพลตฟอร์มใช้ AI ดึงคอนเทนต์สาธารณะมาสร้างใหม่โดยไม่ขอ opt-in ก่อน

MEDIUM Cohere · International

Cohere เผยเทคนิค Hardware-Aware Dynamic Speculative Decoding (DSD) เร่งความเร็ว LLM inference ในระบบ production

Cohere เผยแพร่เทคนิค Hardware-Aware Dynamic Speculative Decoding (DSD) ต่อยอดจาก speculative decoding (SD) ซึ่งใช้โมเดลขนาดเล็ก (draft model) เสนอ token หลายตัวให้โมเดลหลัก (target model) ตรวจสอบพร้อมกัน จุดอ่อนของ SD แบบเดิมคือใช้จำนวน draft token คงที่ (K) ทำให้ช้าลงเมื่อ batch size สูงและกลายเป็น compute-bound DSD แก้ปัญหาด้วยการปรับค่า K แบบไดนามิกตามข้อจำกัดฮาร์ดแวร์ผ่านการวัดค่า goodput คืออัตราส่วนระหว่าง Acceptance Length กับ Inter-Token Latency ผลทดสอบบนโมเดล Command A (dense) พบว่า DSD เร็วกว่า SD แบบคงที่ถึง 23% ที่ batch size 128 และ 256 พร้อมไม่เกิดอาการถดถอยเหมือน SD เดิม ส่วนโมเดล Command A+ (MoE) ให้ผลใกล้เคียงกับ SD ปกติ ทีมงาน Cohere ได้ส่งเทคนิคนี้เข้าสู่โครงการโอเพนซอร์ส vLLM แล้ว พร้อมรองรับ async scheduling และ full CUDA Graph เพื่อใช้งานได้จริงในระบบ production

ผลกระทบ: เทคนิค DSD ช่วยให้ speculative decoding ใช้งานได้จริงในระบบ production ที่มี batch size เปลี่ยนแปลงตลอดเวลา ลดต้นทุนการให้บริการ LLM ได้กว้างขึ้น

MEDIUM The Batch (DeepLearning.AI) · International

DeepSeek เปิดตัว DSpark เทคนิค speculative decoding ใหม่ เร่งความเร็วโมเดล DeepSeek-V4 กว่า 50%

The Batch รายงานว่าทีมวิจัยจาก Peking University และ DeepSeek เปิดตัว DSpark เทคนิค speculative decoding ตัวใหม่ที่ช่วยเร่งความเร็วการสร้างข้อความของโมเดล DeepSeek-V4 ได้มากกว่า 50% โดยไม่เสียความแม่นยำ พร้อมเปิดซอร์สเป็น checkpoint ชื่อ DeepSeek-V4-Pro-DSpark และ DeepSeek-V4-Flash-DSpark ภายใต้สัญญาอนุญาต MIT ที่ใช้เชิงพาณิชย์ได้ จุดเด่นของ DSpark คือการรวมจุดแข็งของ drafter แบบ sequential และ parallel เข้าด้วยกัน ผ่านการเพิ่ม Markov head แก้ปัญหาการเรียงลำดับ token ผิดพลาด และใช้ confidence head พร้อมการปรับเทียบค่าความมั่นใจแบบไดนามิกตามภาระงานของเซิร์ฟเวอร์ ผลทดสอบพบว่า DSpark เพิ่มความเร็วการสร้าง token ต่อผู้ใช้ของ DeepSeek-V4-Flash ได้ 60-85% และ DeepSeek-V4-Pro ได้ 57-78% เมื่อเทียบกับ drafter เดิมของบริษัท ในบางกรณีที่ต้องการความเร็วสูง DSpark เพิ่ม throughput รวมได้สูงถึง 661% ถือเป็นความก้าวหน้าสำคัญด้านการลดต้นทุนการให้บริการ LLM ในระดับ production

ผลกระทบ: DSpark ช่วยลดต้นทุนและเวลาในการให้บริการ LLM ระดับ production ได้อย่างมีนัยสำคัญโดยไม่ต้องแลกกับความแม่นยำ อาจกลายเป็นมาตรฐานใหม่ของเทคนิค speculative decoding

ทีมวิจัยนำโดย Yifan Wu และ Zhuokai Zhao เสนอปัญหาที่เรียกว่า behavioral state decay ซึ่งเกิดขึ้นเมื่อ AI agent ทำงานที่มี long-horizon trajectory ยาวจนข้อมูลสำคัญ เช่น เป้าหมายของงาน ข้อเท็จจริงของสภาพแวดล้อม ความพยายามก่อนหน้า และ subgoal ที่ยังไม่เสร็จ ถูกฝังลึกในหรือหลุดออกจาก context window จนไม่มีผลต่อการตัดสินใจ ทีมจึงออกแบบ memory agent แยกต่างหากที่ทำงานควบคู่กับ action agent เดิมโดยไม่ต้องแก้ไข ตัว memory agent จะอัปเดต structured memory bank จาก trajectory ล่าสุดและตัดสินใจว่าจะแทรก reminder ที่อ้างอิงความจำหรือเงียบไว้ โมดูลนี้ใช้งานแบบ plug-and-play ได้กับ agent harness ที่มีอยู่ ผลทดสอบบน Terminal-Bench 2.0 และ τ²-Bench พบว่าเพิ่ม pass@1 ได้ถึง 8.3 และ 6.8 percentage point ตามลำดับ และการแทรกแบบเลือกเฉพาะจุดให้ผลดีกว่าการเปิดใช้ตลอดเวลาหรือ retrieval ทั่วไป ทีมยังฝึก Qwen3.5-27B ด้วย SFT และ GRPO เพื่อสร้าง open-weight memory policy

ผลกระทบ: เทคนิคนี้ช่วยแก้ปัญหาความจำเสื่อมของ AI agent ในงานที่มี trajectory ยาว ทำให้ agent ทำงานซับซ้อนได้แม่นยำขึ้นโดยไม่ต้องแก้ไขโมเดลหลัก

From Prompts to Contracts: วิศวกรรม Harness เพื่อ AI Agent ระดับองค์กรที่ตรวจสอบได้

นักวิจัย Joongho Ahn และ Moonsoo Kim เสนอแนวทาง harness engineering เพื่อแก้ปัญหาที่แอปพลิเคชัน LLM ระดับองค์กรมักเริ่มต้นจาก prototype ที่พฤติกรรมถูกกำหนดด้วย prompt และ retrieval context ล้วนๆ ทำให้ยากต่อการตรวจสอบเมื่อนำไปใช้งานจริง แนวทางนี้ย้ายพฤติกรรมที่ต้องแน่นอนไปไว้ในโค้ด manifest schema และ validation artifact รอบ composition boundary ที่เปลี่ยนได้ ขณะที่คำตอบที่อ้างอิงแหล่งข้อมูลยังคงเป็นผู้มีอำนาจตัดสินคำตอบสุดท้าย ทีมทดสอบกับข้อมูลสาธารณะของกลุ่มบริษัทเกาหลี 5 กลุ่ม 25 บริษัทจดทะเบียน พบว่า harness รักษาสัญญาด้าน source-grounding entity-routing trace และ output-hygiene ได้ครบถ้วน ผ่านการทดสอบ fault-injection และคงผลเดิมแม้เปลี่ยนโมเดลเป็น 3 โมเดลต่างกันครบ 270 ครั้ง ที่สำคัญคือการบังคับใช้ผ่านโค้ดเท่านั้นที่ป้องกันการรั่วไหลของ internal trace และภาษาแนะนำที่ไม่เหมาะสมได้ครบ ขณะที่ guardrail ภายนอกป้องกันได้เช่นกันแต่ over-refuse จนความสามารถลดจาก 120 เหลือ 88 จาก 120 กรณี

ผลกระทบ: งานวิจัยชี้ว่าองค์กรควรออกแบบ AI agent ให้มีการบังคับใช้กฎผ่านโค้ดแทนการพึ่งพา prompt เพียงอย่างเดียว เพื่อให้ตรวจสอบได้และไม่สูญเสียความสามารถของระบบ

ทฤษฎีปฐมภูมิของการคิดช้าและการรับรู้เชิงรุกในโมเดลภาษาขนาดใหญ่

ทีมวิจัยนำโดย Hongkang Yang ร่วมกับ Weinan E นักคณิตศาสตร์ประยุกต์ชื่อดัง เสนอทฤษฎีทางคณิตศาสตร์ที่อธิบายกลไกการคิดและการรับรู้ของ AI อย่างเป็นระบบ ในฐานะส่วนหนึ่งของงานวิจัยชุดการสร้างแบบจำลองฟังก์ชันทางปัญญาแบบปฐมภูมิ ทฤษฎีนี้อนุมานแนวคิด slow thinking หรือการรับรู้เชิงรุกอย่างเป็นรูปธรรม ครอบคลุมทั้งการออกแบบ การฝึก และการอนุมานของโมเดลภาษาแบบ slow thinking โดยเริ่มจากการยกระดับและฉายภาพการกระจายความน่าจะเป็นระหว่างปริภูมิที่สังเกตได้กับปริภูมิแฝง ทีมเสนอทฤษฎี active lifting ที่อาศัยการสุ่มตัวอย่างลำดับแฝงและแรงขับเคลื่อนภายในเพื่อลดความไม่แน่นอนด้วยอัตราสูงสุด ซึ่งให้พื้นที่การออกแบบขนาดใหญ่ที่ครอบคลุมโมเดล slow thinking เป็นกรณีย่อย ทฤษฎีนี้ยังอนุมานกระบวนการอนุมานที่มีแกนเวลาภายใน และเป้าหมายการฝึกที่คล้ายกับการเข้ารหัสความยาวขั้นต่ำ ผลพลอยได้ทางเทคนิคได้แก่แนวทางสามขั้นตอนในการปรับปรุงโมเดล slow thinking วิธีการรวมสำหรับสร้าง encoder และโมเดล generative ในทุก modality และแนวทางแก้ปัญหา policy collapse ที่เป็นไปได้ งานนี้ตีพิมพ์ใน Journal of Machine Learning ปี 2026

การศึกษาเชิงปฏิบัติของ Relaxed Speculative Decoding แบบไม่ต้องฝึกเพิ่ม

ทีมวิจัยจาก Guoxuan Xia, Luka Ribar และ Paul Balanca ศึกษาเทคนิค relaxed speculative decoding ซึ่งเป็นการปรับปรุงเทคนิค speculative decoding ที่ใช้เร่งความเร็วการ sampling ของ LLM แบบ autoregressive ด้วยโมเดลผู้ช่วยที่เร็วกว่าในการร่างโทเคนก่อนให้โมเดลหลักตรวจสอบแบบขนาน โดยปกติ speculative decoding แบบมาตรฐานเป็นแบบ lossless คือขั้นตอน rejection และ resampling รักษาการกระจายผลลัพธ์เดิมของ LLM ไว้ทุกประการ แต่งานวิจัยล่าสุดชี้ว่าการผ่อนคลายเงื่อนไขนี้อาจให้ความเร็วเพิ่มขึ้น ควบคุม trade-off ระหว่างความสามารถกับความเร็วได้ หรือแม้แต่เพิ่มความสามารถของโมเดล ทีมจึงศึกษาเทคนิคเหล่านี้แบบไม่ต้องฝึกเพิ่ม รวบรวมแนวทางที่มีอยู่เข้าเป็นกรอบเดียวกัน และทดสอบกับสภาพแวดล้อมปัจจุบันเพื่อสรุปข้อค้นพบสำหรับผู้ปฏิบัติงานจริง ข้อสรุปสำคัญคือการผ่อนคลายเงื่อนไขต้องการการประเมินความสามารถอย่างละเอียดต่างจาก speculative decoding แบบ lossless และหลายเทคนิคที่ผ่อนคลายต้องพึ่งพา drafter ที่เป็นโมเดลภาษาคุณภาพดี ทำให้ไม่เหมาะกับ drafter ขนาดเบาแบบ multi-token-prediction

ผลกระทบ: งานวิจัยนี้ให้แนวทางปฏิบัติที่ชัดเจนสำหรับทีมที่ต้องการเร่งความเร็วการ inference ของ LLM ด้วย relaxed speculative decoding โดยชี้ข้อจำกัดสำคัญเรื่องคุณภาพของ drafter model
