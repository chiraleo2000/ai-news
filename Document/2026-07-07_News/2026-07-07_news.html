Andrew Ng แบ่งปัน Loop Design ทั้ง 3 วงจร ลดเวลาจากหลักเดือนเหลือหลักวัน

Andrew Ng ศาสตราจารย์ด้าน AI ชื่อดัง ได้สรุปแนวคิด Loop Engineering ซึ่งเป็นเทรนด์ที่กำลังร้อนแรงในหมู่นักพัฒนา AI Agent ระดับโลก โดยมีจุดเริ่มจากคำพูดของ Boris Cherny หัวหน้าทีม Claude Code แห่ง Anthropic ที่ว่า 'ผมไม่พิมพ์สั่ง Claude เองอีกแล้ว แต่มีลูปคอยสั่งแทน' ก่อนที่ Peter Steinberger เจ้าของโปรเจกต์ OpenClaw และ Addy Osmani จาก Google Chrome จะร่วมกันเรียบเรียงเป็นกรอบความคิดอย่างเป็นทางการ Andrew Ng แบ่งเวิร์กโฟลว์การพัฒนาซอฟต์แวร์ยุค AI ออกเป็น 3 วงจรตามความเร็ว ได้แก่ Agentic Coding Loop ที่ให้ AI เขียนโค้ด ทดสอบ และแก้บั๊กเองทุก 2-3 นาที, Developer Feedback Loop ที่มนุษย์ใช้ความเข้าใจบริบทและอารมณ์ผู้ใช้มาปรับสเปกงาน และ External Feedback Loop ที่นำซอฟต์แวร์ไปทดสอบกับผู้ใช้จริงในตลาด แนวคิดนี้ช่วยลดเวลาการพัฒนาซอฟต์แวร์จากหลักเดือนเหลือเพียงหลักวัน สะท้อนการเปลี่ยนบทบาทของนักพัฒนาจากผู้เขียนโค้ดสู่ผู้ออกแบบระบบวนซ้ำให้ AI ทำงานแทน

Impact: Loop Engineering กำลังเปลี่ยนบทบาทนักพัฒนาซอฟต์แวร์จากผู้เขียนโค้ดเป็นผู้ออกแบบระบบวนซ้ำที่ควบคุม AI Agent ให้ทำงานอัตโนมัติ

รัฐบาลอัลเบอร์ตาใช้ Claude สแกนโค้ด 466 ล้านบรรทัดใน 20 ชั่วโมง อุดช่องโหว่ไซเบอร์ทั่วระบบราชการ

Anthropic เผยกรณีศึกษารัฐบาลรัฐอัลเบอร์ตา ประเทศแคนาดา ซึ่งใช้ Claude Code ร่วมกับโมเดล Opus และ Sonnet ตรวจสอบระบบไอทีของรัฐมาตั้งแต่ปี 2025 ทีมงานในกระทรวงเทคโนโลยีและนวัตกรรมของอัลเบอร์ตาใช้เอเจนต์ Claude ประมาณ 50 ตัวทำงานพร้อมกันแบบอัตโนมัติ สแกนโค้ด 466 ล้านบรรทัดครอบคลุม 1,280 แอปพลิเคชันและ 3,400 repository ภายในเวลาเพียง 20 ชั่วโมง ซึ่งทีมประเมินว่าหากทำด้วยวิธีเดิมจะใช้เวลาราว 6.5 ปี Claude ยังช่วยเขียนและทดสอบแพตช์แก้ไขช่องโหว่ รวมถึงปรับปรุงระบบเก่าที่เขียนด้วยภาษา Java มา 25 ปีให้เสร็จในเวลาเพียง 4-5 วัน นอกจากนี้ยังมีการสร้างเอเจนต์ 'red team' และ 'blue team' บน Claude Agent SDK เพื่อตรวจสอบความปลอดภัยต่อเนื่องตามมาตรฐานสากลกว่า 95 รายการต่อแอปพลิเคชัน อัลเบอร์ตาเผยแพร่ white paper แบ่งปันแนวทางนี้ให้หน่วยงานรัฐอื่นทั่วโลกนำไปประยุกต์ใช้

Impact: กรณีนี้แสดงให้เห็นว่า AI agent สามารถเร่งกระบวนการตรวจสอบและแก้ไขความปลอดภัยไซเบอร์ในระบบราชการขนาดใหญ่ได้เร็วกว่าวิธีเดิมนับร้อยเท่า เปิดทางให้หน่วยงานรัฐอื่นนำแนวทางนี้ไปใช้

Reddit ใช้ LLM ปราบสแปมที่เกิดจาก LLM ด้วยกันเอง

Reddit เปิดเผยว่าได้พัฒนาเครื่องมือที่ใช้โมเดลภาษาขนาดใหญ่ (LLM) เพื่อลดปริมาณสแปมและเนื้อหาบอทบนแพลตฟอร์ม ซึ่งส่วนใหญ่ถูกสร้างขึ้นด้วย LLM เช่นกัน กลายเป็นความย้อนแย้งที่แพลตฟอร์มต้องใช้ AI ต่อสู้กับ AI ตามข้อมูลของ Reddit ระบบใหม่นี้ช่วยบล็อกการเข้าชมสแปมได้ถึง 23 ล้านครั้งต่อวัน และตรวจจับโพสต์และคอมเมนต์สแปมใหม่ได้ประมาณ 25,000 รายการต่อวัน บริษัทระบุว่า LLM ช่วยตรวจจับรูปแบบพฤติกรรมปลอมที่ประสานกันอย่างแนบเนียนซึ่งระบบเดิมตรวจไม่พบ และลดการเห็นสแปมของผู้ใช้ลง 20% เมื่อเทียบช่วงมกราคมถึงมีนาคมกับไตรมาสก่อนหน้า อย่างไรก็ตาม ผู้เชี่ยวชาญด้านการกำกับดูแลเนื้อหาเตือนว่าการตรวจจับด้วย AI ควรควบคู่กับการตรวจสอบโดยมนุษย์เพื่อผลลัพธ์ที่แม่นยำที่สุด

OpenAI เปิดตัว GeneBench-Pro เบนช์มาร์กชีววิทยา 129 ข้อ พบ GPT-5.6 Sol ทำได้เพียง 31.5%

OpenAI เปิดตัว GeneBench-Pro เบนช์มาร์กด้านชีววิทยาเชิงคำนวณจำนวน 129 ปัญหา ครอบคลุมการวิเคราะห์จีโนม โครงสร้างโปรตีน และการออกแบบการทดลองในห้องปฏิบัติการ ผลที่สำคัญที่สุดคือ GPT-5.6 Sol Pro โมเดลที่มีความสามารถสูงสุดที่ OpenAI เคยเปิดให้ทดสอบ ทำคะแนนได้เพียง 31.5% บนเบนช์มาร์กนี้ ขณะที่ Claude Opus 4.8 ทำได้เพียง 16% ตัวเลขเหล่านี้ไม่ใช่ความผิดพลาดเล็กน้อย แต่สะท้อนว่าชีววิทยาเชิงคำนวณระดับผู้เชี่ยวชาญเป็นงานที่ยากกว่าการเขียนโค้ด คณิตศาสตร์ หรือการให้เหตุผลทั่วไปมาก ความรู้ทางวิทยาศาสตร์ระดับปริญญาเอกที่จำเป็นสำหรับ GeneBench-Pro ยังไม่ถูกจับได้ดีในกระบวนการฝึกโมเดล frontier ปัจจุบัน แม้จะเป็นโมเดลขนาดใหญ่อย่าง Fable 5 หรือ GPT-5.6 Sol ก็ตาม เบนช์มาร์กนี้ถูกวางให้เป็นเกณฑ์วัดพัฒนาการของ Claude Science และ GPT-Rosalind ของ OpenAI ในอนาคต และเป็นเครื่องมือสำคัญให้องค์กรประเมินว่า AI ยังต้องการ tooling และ workflow ตรวจสอบเฉพาะทางอีกมากก่อนทดแทนผู้เชี่ยวชาญมนุษย์ในงานวิทยาศาสตร์ที่ยาก

Impact: GeneBench-Pro เผยให้เห็นข้อจำกัดสำคัญของ AI frontier ในงานวิทยาศาสตร์ชีวภาพระดับผู้เชี่ยวชาญ แม้โมเดลที่ล้ำหน้าที่สุดก็ยังทำคะแนนต่ำกว่า 32% ชี้ว่ายังต้องพัฒนาอีกมากก่อนใช้แทนนักวิจัยได้จริง

NVIDIA เปิดตัวงานวิจัย Unified Audio Intelligence โมเดลภาษาที่เข้าใจเสียงได้โดยไม่เสียความสามารถด้านข้อความ

ทีมวิจัยจาก NVIDIA (Nemotron Labs) เผยแพร่ผลงานวิจัยเรื่อง Unified Audio Intelligence ซึ่งพัฒนาโมเดลภาษาแบบรวม (unified) ที่เพิ่มความสามารถด้านการเข้าใจเสียง (audio understanding) โดยไม่ทำให้ความสามารถด้านข้อความ (text intelligence) ถดถอยลง ปัญหาหลักที่งานนี้ต้องการแก้คือเมื่อฝึกโมเดลภาษาให้เข้าใจเสียงเพิ่มเติม มักเกิดการแลกเปลี่ยน (trade-off) ที่ทำให้ความสามารถด้านภาษาเดิมของโมเดลลดลง ทีมวิจัยเสนอวิธีการฝึกและสถาปัตยกรรมที่รักษาสมดุลระหว่างสองโมดัลิตี้นี้ พร้อมเปิดเผยโมเดลที่ได้ในตระกูล Nemotron-Labs-Audex เป็น open-weight บน Hugging Face ให้นักพัฒนานำไปทดลองต่อได้ทันที งานนี้มีความสำคัญเพราะโมเดล audio-text แบบ unified ที่ใช้งานได้จริงในระดับ production ยังเป็นความท้าทายสำคัญของอุตสาหกรรม AI ด้านเสียงในปัจจุบัน

Impact: งานวิจัยนี้ช่วยให้นักพัฒนาสร้างโมเดล multimodal เสียง-ข้อความที่ไม่ต้องแลกความสามารถด้านภาษาเพื่อเพิ่มความเข้าใจเสียง

งานวิจัยเผย LLM เก็บข้อมูล 'ความยาวข้อความที่เหลือ' ไว้แบบเชิงเส้นในตัวโมเดล

งานวิจัยโดย Mohamed Amine Merzouk, Dmitri Carpov, Mirko Bronzi, Damiano Fornasiere และ Adam Oberman พบว่าโมเดลภาษาขนาดใหญ่ (LLM) มีการเข้ารหัสข้อมูลเกี่ยวกับจำนวน token ที่เหลือที่จะสร้างต่อไปไว้ในลักษณะเชิงเส้น (linear encoding) ภายใน hidden representation ของตัวเอง กล่าวคือระหว่างที่โมเดลกำลัง generate ข้อความ มันสามารถ 'รู้' ล่วงหน้าโดยนัยว่าจะต้องสร้างคำอีกกี่ token ก่อนจบประโยคหรือคำตอบ แม้จะไม่ได้ถูกฝึกให้ทำนายค่านี้โดยตรง การค้นพบนี้อยู่ในกลุ่มงานวิจัยด้าน interpretability ที่พยายามเข้าใจกลไกภายในของ LLM งานตีพิมพ์ความยาว 21 หน้า พร้อมภาพประกอบ 9 ภาพ แสดงหลักฐานเชิงประจักษ์จากหลายโมเดล ผลการค้นพบนี้อาจนำไปใช้ปรับปรุงการควบคุมความยาวคำตอบและการตัดสินใจหยุด generate ของโมเดลในอนาคต

Impact: การค้นพบนี้เปิดทางให้พัฒนาเทคนิคควบคุมความยาวคำตอบของ LLM ได้แม่นยำขึ้นโดยใช้สัญญาณภายในของโมเดลเอง

งานวิจัยชี้โมเดลภาษาแทนค่าและแปลง 'แนวคิด' ด้วยโครงสร้างเชิงเรขาคณิตร่วมกัน

Zhimin Hu, Lanhao Niu และ Sashank Varma นำเสนองานวิจัยที่ศึกษาว่าโมเดลภาษาขนาดใหญ่แทนค่า (represent) และแปลง (transform) 'แนวคิด' หรือ concept ต่างๆ ด้วยโครงสร้างทางเรขาคณิต (geometry) ที่มีลักษณะร่วมกันข้าม concept งานนี้อยู่ในสาย mechanistic interpretability ที่พยายามเปิด 'กล่องดำ' ของ LLM เพื่อทำความเข้าใจว่าความรู้และแนวคิดต่างๆ ถูกจัดเก็บและประมวลผลอย่างไรใน hidden space ของโมเดล ผลการค้นพบว่ามี geometry ร่วมกันบ่งชี้ว่า LLM อาจมีกลไกการจัดระเบียบความรู้ที่เป็นระบบและคาดเดาได้มากกว่าที่เคยคิด ซึ่งเป็นก้าวสำคัญสำหรับการพัฒนาเครื่องมือ interpretability และ debugging โมเดลในอนาคต งานวิจัยนี้เผยแพร่ในหมวด cs.CL และ cs.AI บน arXiv วันที่ 7 กรกฎาคม 2026

Impact: การค้นพบโครงสร้างเรขาคณิตร่วมของแนวคิดใน LLM ช่วยวางพื้นฐานสำหรับเครื่องมือ interpretability ที่แม่นยำขึ้นในอนาคต

อีก 9 วันกฎหมายควบคุมบริการ AI ที่เลียนแบบบุคลิกภาพมนุษย์ของจีนจะมีผลบังคับใช้ ทำให้ Doubao ของ ByteDance และ Qwen ของ Alibaba ต้องปิดฟีเจอร์เอเจนต์แบบมนุษย์และเอเจนต์ที่ผู้ใช้สร้างเองก่อนถึงกำหนด กฎหมายซึ่งออกร่วมโดยสำนักบริหารไซเบอร์สเปซของจีนและหน่วยงานอีก 4 แห่งเมื่อเมษายน 2026 กำหนดให้บริการ AI ที่เลียนแบบบุคลิกภาพต้องมีระบบป้องกันการเสพติด การแจ้งเตือนการใช้งาน และกลไกออกจากระบบทันที Doubao ซึ่งมีผู้ใช้กว่า 345 ล้านคนต่อเดือน จะปิดฟีเจอร์เอเจนต์วันที่ 15 กรกฎาคม โดยให้ผู้ใช้ดูข้อมูลย้อนหลังได้จนถึง 15 ตุลาคม ก่อนข้อมูลจะถูกลบถาวร ส่วน Qwen ของ Alibaba สถานการณ์รุนแรงกว่าเพราะไม่มีช่องทางย้ายข้อมูลให้ผู้ใช้เลย ทั้งสองบริษัทเลือกปิดฟีเจอร์ทั้งหมดแทนการปรับสถาปัตยกรรมให้สอดคล้องกฎหมาย เพราะกลไกป้องกันการเสพติดขัดแย้งโดยธรรมชาติกับการออกแบบเอเจนต์แบบจดจำข้อมูลต่อเนื่อง

Impact: กฎระเบียบของจีนสร้างข้อจำกัดเชิงโครงสร้างที่ขัดแย้งกับการออกแบบ AI agent แบบ persistent memory ส่งผลให้ผู้เล่นรายใหญ่ต้องปิดฟีเจอร์ทั้งหมดแทนการปรับตัว ซึ่งอาจเป็นบรรทัดฐานให้ประเทศอื่นพิจารณากฎหมายคล้ายกัน
