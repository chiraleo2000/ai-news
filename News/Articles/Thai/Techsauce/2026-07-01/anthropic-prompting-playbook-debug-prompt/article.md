## สรุปข่าว

ในงาน Code with Claude เซสชัน The Prompting Playbook คุณ Margot van Laar วิศวกรจาก Anthropic ประจำลอนดอน สาธิตวิธีแก้ปัญหา Prompt ที่ใช้งานได้ดีในโปรดักชันแต่กลับพังหลังย้ายไปใช้โมเดลใหม่ที่เก่งกว่าเดิม โดยชี้ว่าต้องเริ่มจากการสร้างชุดทดสอบประเมินผลหรือ Eval เพื่อวัดผลกระทบของการแก้ Prompt อย่างเข้มงวด จากนั้นเก็บกวาดโครงสร้าง Prompt ด้วยแท็ก XML แยกบทบาท นโยบาย และโทนการพูดให้ชัดเจน ก่อนไล่แก้ปัญหาเฉพาะจุด เช่น การที่โมเดลหวงข้อมูลที่เข้าถึงได้ การให้เครื่องมือคำนวณแทนให้โมเดลคิดเลขในใจ และการเล่าข้อแลกเปลี่ยนทั้งสองด้านแทนการสั่งห้ามเพียงด้านเดียว บทเรียนสำคัญคือคำสั่งไม่ได้เพิ่มความสามารถให้โมเดล และยิ่งโมเดลฉลาดขึ้นต้องให้ข้อมูลที่สมดุลมากขึ้นเพื่อให้ตัดสินใจได้ถูกต้อง

## Tech Insights

**ผลกระทบ:** ให้แนวทางปฏิบัติที่เป็นรูปธรรมสำหรับวิศวกร AI ในการดูแลและแก้ไข Prompt เมื่อต้องย้ายไปใช้โมเดลใหม่
**ระดับความสำคัญ:** medium

### นักพัฒนา/Engineer ควรทำอะไร
- สร้างชุด Eval ที่ครอบคลุม Control case, Edge case และขอบเขตความสามารถของโมเดลก่อนแก้ Prompt ใดๆ
- ใช้แท็ก XML จัดโครงสร้าง Prompt แยกบทบาท นโยบาย และโทนการพูดให้ชัดเจนเพื่อลดความสับสนของโมเดล

### ธุรกิจ/องค์กร ควรทำอะไร
- วางระบบควบคุมเวอร์ชันสำหรับ Prompt ที่ใช้ในโปรดักชัน พร้อมบันทึกเหตุผลของคำสั่งเชิงป้องกันแต่ละข้อ
- ประเมินความเสี่ยงก่อนย้ายระบบ AI ไปใช้โมเดลใหม่ โดยเทียบผลจาก Eval แทนความรู้สึกว่าน่าจะดีขึ้น
