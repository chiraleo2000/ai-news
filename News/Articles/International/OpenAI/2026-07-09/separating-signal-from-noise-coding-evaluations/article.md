# OpenAI เผยผลตรวจสอบพบ SWE-Bench Pro ราว 30% ของงานมีปัญหา ถอนคำแนะนำให้ใช้เป็นตัวชี้วัดมาตรฐาน

## สรุปเนื้อหา

OpenAI เผยแพร่ผลการตรวจสอบเชิงลึกต่อ SWE-Bench Pro ซึ่งเป็นหนึ่งใน benchmark ที่ใช้วัดความสามารถด้าน agentic coding ของโมเดล AI ที่ได้รับความนิยมมากที่สุด หลังจากก่อนหน้านี้เคยแนะนำให้ชุมชนเปลี่ยนมาใช้ SWE-Bench Pro แทน SWE-bench Verified ที่พบปัญหาการปนเปื้อนข้อมูล ทีมวิจัยใช้ pipeline ตรวจสอบข้อมูลอัตโนมัติร่วมกับ agent ตรวจสอบที่ใช้ Codex และแคมเปญให้วิศวกรซอฟต์แวร์มืออาชีพ 5 คนต่อหนึ่งงานช่วยตรวจซ้ำ ผลพบว่า pipeline อัตโนมัติตั้งข้อสงสัยกับ 200 งาน (27.4%) ว่ามีปัญหา ขณะที่การตรวจสอบโดยมนุษย์ระบุว่ามีถึง 249 งาน (34.1%) ที่ชำรุด ปัญหาหลักมี 4 ประเภทคือ การทดสอบที่เข้มเกินไป โจทย์ที่ระบุไม่ครบถ้วน การทดสอบที่ครอบคลุมไม่พอ และโจทย์ที่ทำให้เข้าใจผิด OpenAI จึงประกาศถอนคำแนะนำให้ใช้ SWE-Bench Pro และเรียกร้องให้ชุมชนพัฒนา benchmark ใหม่ที่สร้างโดยวิศวกรผู้มีประสบการณ์เพื่อความน่าเชื่อถือในการวัดศักยภาพโมเดล

## ข้อมูลเชิงเทคนิคและผลกระทบ

**ผลกระทบ:** การค้นพบนี้ชี้ว่า benchmark ด้านการเขียนโค้ดที่ใช้กันแพร่หลายอย่าง SWE-Bench Pro มีความน่าเชื่อถือต่ำกว่าที่คาด ส่งผลต่อการตัดสินใจด้านความปลอดภัยและการพัฒนาโมเดลทั่วอุตสาหกรรม

**สิ่งที่นักพัฒนาควรทำ:**
- ทีมพัฒนาที่ใช้ SWE-Bench Pro วัดผลโมเดลควรระงับการอ้างอิงตัวเลข pass rate ชั่วคราว และตรวจสอบผลลัพธ์ด้วยชุดข้อมูลอื่นควบคู่กัน
- ศึกษาแนวทาง pipeline ตรวจสอบคุณภาพข้อมูลแบบ agent-assisted ของ OpenAI เพื่อนำไปประยุกต์ตรวจสอบ benchmark หรือชุดทดสอบภายในองค์กรของตนเอง

**สิ่งที่ธุรกิจควรทำ:**
- องค์กรที่ใช้ผลการเทียบเบนช์มาร์กโค้ดเพื่อเลือกโมเดล AI ควรพิจารณาความเสี่ยงจากข้อมูลที่ไม่น่าเชื่อถือ และรอ benchmark ใหม่ที่ผ่านการตรวจสอบเข้มงวดกว่า
- ติดตามพัฒนาการของ benchmark ทางเลือกที่ชุมชนหรือ OpenAI จะเสนอ เพื่อประกอบการประเมินผู้ให้บริการ AI coding assistant ในอนาคต
