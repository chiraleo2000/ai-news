# LLM เข้ารหัสความยาวคำตอบที่เหลือไว้แบบเชิงเส้นในเลเยอร์ซ่อน ก่อนเริ่มสร้างคำตอบด้วยซ้ำ

**แหล่งที่มา:** [arXiv:2607.05316](https://arxiv.org/abs/2607.05316) | Mohamed Amine Merzouk, Dmitri Carpov, Mirko Bronzi, Damiano Fornasiere, Adam Oberman | ส่งเมื่อ 6 ก.ค. 2026

## สรุปสั้น

งานวิจัยพบว่าโมเดลภาษาขนาดใหญ่มี "แผน" ภายในเกี่ยวกับความยาวคำตอบที่จะสร้าง โดยสามารถถอดรหัสความยาวคำตอบทั้งหมดได้แบบเชิงเส้นจาก hidden state ของ prompt ก่อนที่จะเริ่มสร้างคำตอบเลย

## สรุปงานวิจัย

โมเดลภาษาสร้างคำตอบทีละ token แต่คำตอบที่ได้กลับมีโครงสร้างความยาวที่สม่ำเสมออย่างน่าประหลาด เช่น การแก้ปัญหาแบบทีละขั้นมักลงเอยด้วยจำนวน token ที่คาดเดาได้ การค้นคำตอบมักหยุดหลังไม่กี่ประโยค และการถอยกลับแก้ไขคำตอบทำให้คำตอบยาวขึ้นในปริมาณที่วัดได้ ทีมวิจัยจึงตั้งคำถามว่าโมเดลมีการประมาณค่าภายในว่าคำตอบที่เหลือยังมีอีกเท่าไรหรือไม่ พวกเขาฝึก linear probe ขนาดเล็กบน hidden state ที่แช่แข็งของโมเดล open-weight ขนาด 7-8B จำนวนสามตัว บนชุดข้อมูล completion เจ็ดชุด

## ผลการทดลองหลัก

- **ถอดรหัสได้ก่อนเริ่มตอบ**: ความยาวคำตอบทั้งหมดถอดรหัสได้แบบเชิงเส้นจาก hidden state สุดท้ายของ prompt เพียงอย่างเดียว โดยยังไม่มี token เอาต์พุตใดถูกสร้างขึ้นเลย
- **ถ่ายโอนได้แบบไม่สมมาตร**: ทิศทางของ probe ที่ฝึกจากข้อมูลภาษาธรรมชาติสามารถถ่ายโอนไปยังชุด synthetic completion ที่ไม่เคยเห็นได้ดีกว่า baseline ทางสถิติ ในขณะที่ทิศทางย้อนกลับ (จาก synthetic ไปภาษาธรรมชาติ) มักล้มเหลว ความไม่สมมาตรนี้เองก็มีความหมายในตัว
- **ตอบสนองต่อการถอยกลับ**: บนตัวอย่าง high-loss completion ที่โมเดลถอยกลับและเริ่มแก้ปัญหาใหม่ ค่าประมาณความยาวที่เหลือของ probe จะขยับสูงขึ้นทันทีที่เกิดการถอยกลับ ซึ่งตัวทำนายที่ใช้แค่ตำแหน่ง token ไม่สามารถทำได้

ทีมวิจัยเน้นว่าผลลัพธ์นี้เป็น "การประมาณค่าความยาวเอาต์พุตที่เหลือ" ซึ่งต่างจาก impossibility result ของการนับ exact-length ใน transformer และตีความว่าโมเดลมีการแทนค่าภายในแบบมีแผน (plan-like) สำหรับความยาวเอาต์พุต แม้ยังไม่พิสูจน์ว่าถูกใช้เชิงเหตุ-ผลจริงในการสร้างคำตอบ

## ผลกระทบต่อวงการ AI

การค้นพบนี้เพิ่มความเข้าใจด้าน mechanistic interpretability ของ LLM และอาจนำไปสู่เครื่องมือควบคุมความยาวคำตอบ ตรวจจับพฤติกรรมถอยกลับแก้ไข หรือคาดการณ์ต้นทุน inference ล่วงหน้าได้แม่นยำขึ้น

## แนวทางปฏิบัติ

**สำหรับนักพัฒนา:**
- ทดลองฝึก linear probe บน hidden state ของโมเดลที่ทีมใช้งานเพื่อตรวจสอบพฤติกรรมการวางแผนความยาวคำตอบ
- ใช้แนวคิดนี้พัฒนาเครื่องมือควบคุมความยาวคำตอบหรือตรวจจับการถอยกลับแก้ไขคำตอบแบบเรียลไทม์

**สำหรับธุรกิจ:**
- ประเมินการนำเทคนิคนี้ไปช่วยลดต้นทุน inference โดยควบคุมความยาวคำตอบให้เหมาะสมล่วงหน้า
- ติดตามงานวิจัยด้าน interpretability ที่เกี่ยวกับการวางแผนภายในของ LLM เพื่อประกอบการตัดสินใจเลือกโมเดล
